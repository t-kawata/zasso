# Release v0.24.243

## Summary
Implement Phase 0 of the process-registry crate: pure data types, error types, and process state/registry type definitions with full serde support and unit test coverage.

## What's Changed
- **feat: process-registry crate foundation** — New `crates/procreg/` crate with pure data types (`ProcessDef`, `RestartPolicy`, `ReadyCondition`, `ShutdownTimeoutConfig`), error types (`RegistryError` with `thiserror` + `anyhow`), and process state machine (`ProcessState` with serde JSON round-trip, `RegistryEntry`, `RegistryInner`, `ProcessRegistry`)
- **test: 32 unit tests** — Complete test coverage for all types: field access, Clone semantics, PartialEq equality, serde round-trip for all 6 ProcessState variants, Debug/Display formatting
- **chore: add dependencies** — `thiserror`, `anyhow`, `serde`(derive), `tokio`(sync), `tokio-util`(rt) for the new crate
- **docs: M0 ticket specs and artifacts** — Implementation specs, plans, and review reports for tickets M0-1, M0-2, M0-3
- **docs: update ticket tracking** — Mark completed tickets in Tickets.md and queue.md
