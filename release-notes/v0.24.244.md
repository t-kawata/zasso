# Release v0.24.244

## Summary
Complete Phase 0 of process-registry with DAG topological sort and RestartPolicy methods, and begin Phase 1 with the ChildGuard graceful shutdown implementation.

## What's Changed
- **feat: RestartPolicy methods** — `on_crash_default()` constructor and `next_delay()` exponential backoff calculation with full test coverage (closes M1-1)
- **feat: DAG topological sort** — `resolve_start_order()` using `petgraph::DiGraph` + `toposort` with circular/unknown dependency detection (closes M2-1, completes Phase 0)
- **feat: ChildGuard graceful shutdown** — Replace stub with real implementation wrapping `tokio::process::Child`. Unix: SIGTERM→wait→SIGKILL, Windows: TerminateProcess, Drop: best-effort `start_kill()` (closes M3-1, begins Phase 1)
- **chore: add dependencies** — `petgraph`, `tokio`(process, time, rt), `libc`(cfg(unix))
- **test: 51 unit tests** — All tests passing in 0.00s across all modules
- **docs: ticket specs and artifacts** — M1-1, M2-1, M3-1 specs, implementation summaries, and review reports
