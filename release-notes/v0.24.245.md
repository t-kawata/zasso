# Release v0.24.245

## Summary
Complete Phase 1 and advance Phase 2 of process-registry: platform process liveness check, async ready-condition waiting, ProcessRegistry basic API, and event-driven watch loop with restart policy integration.

## What's Changed
- **feat: is_process_alive** — Cross-platform process existence check. Unix: `libc::kill(pid,0)` + ESRCH. Windows: `OpenProcess` + `CloseHandle`. PID 0 guard (closes M4-1, completes Phase 1)
- **feat: wait_ready** — Async wait for all 4 `ReadyCondition` variants (Immediate, Delay, LogContains via broadcast subscribe, TcpPort via connect polling) with `tokio::time::timeout` for each (closes M5-1, begins Phase 2)
- **feat: ProcessRegistry basic API** — `new()`, `snapshot()`, `subscribe_output()`, `pipe_output_to()` methods on ProcessRegistry (closes M6-1)
- **feat: watch_loop** — Event-driven process monitoring with `tokio::select!` (exit_rx vs cancel_token). RestartPolicy integration with backoff delay. Restart path stubbed awaiting M8-1 (closes M7-1)
- **chore: add dependencies** — `tokio`(net) for TcpStream, `windows`(cfg(windows)) for OpenProcess
- **test: 70 unit tests** (67 active, 3 ignored pending M8-1) — All tests pass in 0.01s
- **docs: ticket specs and artifacts** — M4-1, M5-1, M6-1, M7-1 specs, implementation summaries, and review reports
