# Release v0.24.246

## Summary
Complete Phase 2 and begin Phase 3 of process-registry: process spawning with output capture, dependency-ordered startup, graceful shutdown, Unix SIGTERM handler, and Windows platform verification.

## What's Changed
- **feat: spawn_one** — Real process spawning via `tokio::process::Command` with stdout/stderr capture (broadcast), PID probe, ChildGuard wrapping, and ReadyCondition wait (closes M8-1)
- **feat: start_all** — Dependency-ordered startup using `resolve_start_order`, sequential `spawn_one`, and `watch_loop` attachment (closes M8-1)
- **feat: shutdown_all / stop** — Reverse-order shutdown with CancellationToken cancel + ChildGuard graceful shutdown. Deadlock-safe by releasing Mutex lock before `.await` (closes M9-1, completes Phase 2)
- **feat: Unix SIGTERM handler** — `install_sigterm_handler` catches SIGTERM via `tokio::signal::unix` and calls `shutdown_all()` for clean child process termination (closes M10-1)
- **feat: Windows platform** — Verification of existing `#[cfg(windows)]` implementations (graceful_shutdown + is_process_alive), already completed in M3-1/M4-1 (closes M10-2)
- **test: 75 unit tests** — All tests passing in 0.02s, no ignored tests remaining
- **fix: stopped_state_exits** — Resolved oneshot timing issue by mirroring production execution order (spawn → delay → send → await)
- **chore: add dependencies** — `tokio`(io-util, signal)
- **docs: ticket specs and artifacts** — M8-1, M9-1, M10-1, M10-2 specs, implementation summaries, and review reports
