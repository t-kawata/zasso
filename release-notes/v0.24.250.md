# Release v0.24.250

## Summary

Introduces EDITION_HOME (OnceLock-cached edition home directory) and automatic bifrost binary deployment with version marker detection. Adds project-wide Zero Tolerance rule for quality checker findings. Restructures edition data directories under `.zasso/`.

## What's Changed

- **feat**: Add EDITION_HOME with OnceLock caching — `init_edition_home()` and `edition_home()` available from anywhere via `consts::edition_home()`
- **feat**: Add bifrost binary auto-deployment — bundled tar.gz is extracted to `EDITION_HOME/bifrost/` with version marker (`.version`) for update detection
- **feat**: Add `bifrost/deploy.rs` with `ensure_bifrost_binary()` — creates directory, extracts archive, sets executable permissions, writes version marker
- **feat**: Add `bifrost/assets.rs` with `include_bytes!` for OS-specific binary bundling (macOS ARM64, Linux x86_64, Windows x86_64)
- **feat**: Restructure edition data directories — `mycute` → `.zasso/mycute`, `zasso` → `.zasso/zasso`, `neco-asovi` → `.zasso/neco-asovi`
- **chore**: Add `dirs`, `flate2`, `tar` dependencies; add `tempfile` as dev-dependency
- **chore**: Add `.DS_Store` to `.gitignore`
- **docs**: Add Zero Tolerance rule for `run-quality-checks.js` findings to CLAUDE.md
- **fix**: Eliminate all `expect()`/`unwrap()` calls in production code (both `lib.rs` entry point and `edition_home()`)
- **test**: Add 5 unit tests covering EDITION_HOME lifecycle and bifrost deployment (first deploy, version match skip, version mismatch re-extract, platform-specific filename)
