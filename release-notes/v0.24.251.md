# Release v0.24.251

## Summary

Complete process-registry Fate Sharing implementation — sidecar lifecycle management with cross-platform parent/child death detection, port conflict safety, and unified watchdog wrapper for macOS/Linux/Windows.

## What's Changed

- **feat**: Add process-registry integration in Tauri setup hook — sidecars are now started declaratively via ProcessRegistry with automatic dependency resolution (#26)
- **feat**: Add port conflict detection via `is_port_free()` using `TcpListener::bind` — prevents new app startup when zombie processes hold ports (#27)
- **feat**: Add parent process death monitoring via watchdog wrapper — separate binary monitors parent PID, kills children when parent dies, works on all platforms (#29)
- **feat**: Add child permanent death cascade — when retry limit is exhausted or restart spawn fails, `shutdown_all()` is called to terminate the entire application (#30)
- **refactor**: Remove platform-specific pdeathsig (`prctl`) and `install_parent_monitor()` thread — replaced by unified watchdog (#29)
- **refactor**: Remove `greet` Tauri template command and `#[allow(dead_code)]` workarounds (#26)
- **docs**: Add `consts/settings.rs` configuration constant convention to CLAUDE.md and rust coding-style rules
- **test**: Add 9 new tests covering port conflict detection, watchdog env var passthrough, watchdog binary extraction
- **chore**: Suppress intentional dead_code warnings for frontend-only constants in settings.rs and bifrost assets.rs
