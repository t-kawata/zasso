# Release v0.24.257

## Summary
Added async startup mode to process-registry crate (`start_all_async` + `StartupMonitor`), enabling non-blocking sidecar process launch with timeout-based fate sharing. Zasso now launches bifrost asynchronously without blocking the Tauri setup hook.

## What's Changed
- **feat(procreg)**: Add `start_all_async()` API and `StartupMonitor` for non-blocking sidecar startup with level-based parallel spawning and configurable timeout
- **feat(procreg)**: Add `resolve_start_levels()` for dependency-graph level grouping (`Vec<Vec<String>>`)
- **feat(procreg)**: Add cancel-token awareness to `spawn_one` — `wait_ready` now supports cancellation via `CancellationToken`
- **feat(procreg)**: Add `RegistryError::SpawnCancelled` and `RegistryError::StartupTimeout` error variants
- **feat(zasso)**: Switch sidecar startup from `start_all()` to `start_all_async()` with 30s total timeout and background monitoring
- **perf(zasso)**: Tauri window now appears immediately without waiting for bifrost's `wait_ready` to complete
- **refactor(procreg)**: Make `ChildGuard` cloneable via `Arc<Mutex<Option<Child>>>` for future early-registration support
- **docs(procreg)**: Update README with async startup tutorial, updated architecture diagram, and new error variants (8 total)
