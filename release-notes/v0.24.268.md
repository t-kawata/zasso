# Release v0.24.268

## Summary

Added macOS and Windows native speech recognition backends. Both backends include proper C stub libraries generated at build time for development without native helper libraries. Fixed test tooling to verify backend initialization at runtime.

## What's Changed

- **feat(voiput)**: Added `MacSpeechBackend` (M4-3) — macOS native speech recognition with Tahoe (macOS 15+) and Classic (SFSpeechRecognizer) engine detection. Coalescing/watermark/PostCorrection pipeline in background ticker task
- **feat(voiput)**: Added `WinSpeechBackend` (M4-4) — Windows native speech recognition with IME control, PunctuationMachine integration, health check, and timeout-based punctuation insertion
- **fix(voiput)**: Fixed macOS stub library generation in `build.rs` — replaced empty archive with proper C stub implementing all 15 FFI symbols (link error → `speech_helper_init` returns -1 at runtime)
- **fix(voiput)**: Fixed Windows stub library generation — C stub with all 12 FFI symbols compiled via cl.exe + lib.exe
- **fix(voiput)**: `test_macos()` now actually calls `MacSpeechBackend::new()` instead of always skipping — shows initialization result from stub or real library
- **refactor(voiput)**: macOS backend extracted coalescing/watermark as pure testable functions (9 unit tests)
- **refactor(voiput)**: Windows backend extracted coalescing/watermark/has_unconfirmed as pure testable functions (9 unit tests)
- **chore**: 2 tickets completed and reviewed (M4-3, M4-4)
