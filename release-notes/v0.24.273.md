# Release v0.24.273

## Summary
Phase 6 implementation: Voiput API async/await compliance, permission management, and build system fix for real native library on macOS 16+.

- **feat: M7-1 Voiput async/await compliance** — `start()`, `stop()`, `set_engine()` converted to async fn; `request_permissions()` added for macOS (FFI), Windows (health_check bit 2), and unsupported OS
- **fix: build.rs real library activation** — Removed Swift Concurrency dylib file-existence gate that falsely blocked real libSpeechHelper.a on macOS 16+ (dyld shared cache); restructured to always configure Swift runtime paths
- **fix: test-run.rs VAD model paths** — Replaced dummy `/tmp/` model paths with `model_path()` helper resolving to actual `models/` directory; eliminated Sherpa ONNX validation errors
- **docs: RFC document update** — Added §13 (Phase 6), §14 (Phase 7), 付録G (audit discrepancy table), and table of contents
- **docs: ticket system** — Created ticket #61 (RFC doc fix) and #62 (M7-1: async/request_permissions) with specs, implementation, and review artifacts
