# Release v0.24.282

## Summary
Fixed the Windows prebuilt library linking issue where the stub .lib (2,618 bytes) was used instead of the real Native AOT import library (5,478 bytes), causing speech_helper_init to always return -1. Also eliminated the spurious "Stop Warning" from the C# backend and added comprehensive development environment setup documentation.

## What's Changed
- **fix**: Windows prebuilt library — Native AOT .lib now correctly linked instead of stub. .csproj updated to copy the import library to the publish output directory
- **fix**: build.rs — added fallback .lib copy from subdirectory in try_build_windows_native(); cleaned up stale M6-1 references
- **fix**: C# SpeechHelper.cs — StopInternal() now checks _isCapturing before calling StopAsync(), eliminating the "Stop Warning" on already-completed sessions
- **fix**: test-run.rs — removed misleading M6-1.6 milestone reference from Windows error message
- **docs**: added HOWTO_BUILD_DEV_ENV.md covering Windows, macOS, and Linux development environment setup, including HVCI/Memory Integrity workaround instructions
- **refactor**: prebuilt/windows/ — replaced stub .lib with Native AOT .lib; removed old stub files (speech_helper.lib, speech_helper.exp)
- **chore**: ticket #63 — spec, implementation, and review artifacts for Windows stub removal
