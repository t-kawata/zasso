# Release v0.24.285

## Summary

Added hotkey monitoring module (M8-1) to voiput crate with macOS CGEventTap and Windows rdev/GetAsyncKeyState/WH_KEYBOARD_LL implementations, plus `--hotkeys` test mode for test-run.

## What's Changed

- **feat**: Add `hotkey/` module to voiput crate — `HotkeyAction` enum (Start, BufferFlush, OrchestratorInput, Correct, Summarize), `HotkeyMonitor` with platform-specific implementations
- **feat**: macOS — CGEventTap-based Option double-tap detection, Ctrl+Option orchestator combo, Option+H/Option+M hotkey combos
- **feat**: Windows — rdev listener + GetAsyncKeyState polling dual-path Alt double-tap detection, Ctrl+Alt orchestator combo, WH_KEYBOARD_LL low-level hook
- **feat**: Add `--hotkeys` argument to test-run.rs for live hotkey action monitoring
- **chore**: Add `rdev` dependency for Windows, expand `winapi` features (winuser, libloaderapi, processthreadsapi)
- **test**: 15 new unit tests covering hotkey parsing, bit operations, double-tap boundary, state transitions (131 total passing)
