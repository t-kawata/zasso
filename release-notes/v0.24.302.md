# Release v0.24.302

## Summary

Fixed PostCorrection (LLM post-correction) pipeline — corrected stale pending-clear logic, passthrough watermark corruption, and is_post_correcting deadlock. Made OpenAI API key mandatory for all engine modes. Added env_logger and improved test-run display for post-correction visibility.

## What's Changed

- **feat**: Split post_correction_openai_config from openai_config in VoiputConfig (#81) — enables PostCorrection with --engine os
- **feat**: Add env_logger dependency for runtime log visibility
- **feat**: Improve test-run.rs event display — PostCorrectionStarted/Finished labels, correction indicator
- **fix**: PostCorrectionProcessor is_pending_correction no longer reset by trailing "." events from Tahoe
- **fix**: mac.rs/win.rs ticker passthrough path — restored `if !has_processor` guard to prevent watermark corruption (MYCUTE-compatible behavior)
- **fix**: voiput.rs BufferFlush no longer blocked by is_post_correcting flag (was causing deadlock)
- **fix**: test_voiput() now uses build_voiput_config(args) to include PostCorrection config in test
