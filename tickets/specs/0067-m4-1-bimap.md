# M4-1: `BiMap<RuntimeId, NativeId>` 実装

## Summary

RuntimeId → NativeId および NativeId → RuntimeId の双方向 O(1) マッピングを提供する `BiMap<L, R>` 汎用データ構造を実装する。PJSUA が再利用するネイティブ ID から利用者を保護し、crate 内部で一貫した ID 管理を実現する。

**参照設計書:** [docs/rust-sip-client-rfc.md](../docs/rust-sip-client-rfc.md) (§9, §45.4)

## Background

### RFC 準拠

RFC §9 で「PJSUA の `pjsua_acc_id` や `pjsua_call_id` は再利用されうるため、そのまま公開しない。内部では `BiMap<RuntimeId, NativeId>` で変換する」と規定されている。§45.4 でも「public id を別採番し bi-map 変換」が言及されている。

### 既存チケットからの依存関係

- `AccountId` / `CallId` / `AudioSourceId`（M0-2: #53）— 型パラメータとして使用可能な Hash + Eq + Clone を実装済み
- `util/mod.rs` と `util/id.rs`（M0-2）— 同一 util モジュールに新規ファイル追加

### 後続チケットとの関係

| チケット | 使用箇所 |
|----------|----------|
| M10-2 | MockBackend 内部の ID 管理 |
| M12-1 | SipClient / ClientInner の ID 変換 |
| M12-4 | add_account / remove_account での ID 管理 |

### 設計判断

- **新規ファイル `src/util/bimap.rs`**: `id.rs` は既に 421 行のため、BiMap は別ファイルとして util モジュールに追加する
- **ジェネリクス**: `BiMap<L, R>` は型パラメータ L, R に `Hash + Eq + Clone` を要求する
- **内部実装**: 2 つの `HashMap`（`left_to_right` / `right_to_left`）で双方向マッピングを実現
- **insert の置換動作**: 既存の left または right が衝突した場合、旧ペアを `Some((L, R))` で返し、新ペアに置き換える（upsert 動作）。`None` は新規挿入を示す
- **remove の戻り値**: 削除されたペアを `Option<(L, R)>` で返す。存在しないキーの場合は `None`
- **`#[must_use]`**: `insert` の戻り値は `#[must_use]` を付与し、置換の検出漏れを防止する
- **テスト容易性**: 型パラメータに `u32` と `u64` を使用し、具体的な ID 型に依存しないテストを実現

## Scope

### `crates/siprs/src/util/bimap.rs`（新規）

```rust
use std::collections::HashMap;
use std::hash::Hash;

/// 双方向マッピング。
///
/// `L → R` および `R → L` の両方向のマッピングを O(1) で提供する。
/// PJSUA のネイティブ ID（再利用可能）とランタイム ID（一意）の変換に使用する。
#[derive(Debug, Clone)]
pub struct BiMap<L, R> {
    left_to_right: HashMap<L, R>,
    right_to_left: HashMap<R, L>,
}

impl<L, R> BiMap<L, R>
where
    L: Hash + Eq + Clone,
    R: Hash + Eq + Clone,
{
    /// 空の BiMap を生成する。
    pub fn new() -> Self;

    /// left と right のペアを挿入する。
    ///
    /// 既存の left または right が衝突した場合、旧ペアを `Some((L, R))` で返し、
    /// 新しいペアで置き換える。新規挿入の場合は `None` を返す。
    #[must_use]
    pub fn insert(&mut self, left: L, right: R) -> Option<(L, R)>;

    /// left に対応する right を取得する。
    pub fn get_right(&self, left: &L) -> Option<&R>;

    /// right に対応する left を取得する。
    pub fn get_left(&self, right: &R) -> Option<&L>;

    /// left が存在するか確認する。
    pub fn contains_left(&self, left: &L) -> bool;

    /// right が存在するか確認する。
    pub fn contains_right(&self, right: &R) -> bool;

    /// left に対応するペアを削除する。
    pub fn remove_by_left(&mut self, left: &L) -> Option<(L, R)>;

    /// right に対応するペアを削除する。
    pub fn remove_by_right(&mut self, right: &R) -> Option<(L, R)>;

    /// エントリ数を返す。
    pub fn len(&self) -> usize;

    /// 空かどうか返す。
    pub fn is_empty(&self) -> bool;
}
```

### `crates/siprs/src/util/mod.rs`（修正）
